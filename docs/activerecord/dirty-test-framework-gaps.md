# dirty.test.ts — deferred framework gaps

Surfaced while porting `dirty.test.ts` to a faithful, DDL-free mirror of Rails
`activerecord/test/cases/dirty_test.rb` (PR #2913). That PR replaced ~33
`expect(true).toBe(true)` stubs with 29 real passing tests + honest `it.skip`s.
Each skip below names a real gap; closing it un-skips the listed test(s). These
are **separate implementation PRs** — they touch shared models / core
dirty-tracking and shouldn't ride a test rewrite.

Ordered by yield (tests un-skipped per fix).

## Impossible in JS/Ruby (can never match without stubs)

| Test                                                                                                                                             | Why                                                                                                                                                                                                    |
| ------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `attribute will change!`, `in place mutation detection`, `in place mutation for binary`, `mutating and then assigning doesn't remove the change` | Rails mutates a string in place (`catchphrase << " matey!"`); JS strings are immutable.                                                                                                                |
| `changes is correct if override attribute reader`, `getters with side effects are allowed`                                                       | Rails defines a singleton method on one instance (`def obj.catchphrase ...`); no per-instance method-with-super override in JS.                                                                        |
| `string attribute should compare with typecast symbol after update`                                                                              | Tests that a Ruby symbol (`:foo`) passed to `create!`/`update_column` casts to `"foo"` and so isn't dirty. JS has no auto-coercing symbol; substituting `"foo"` would test `"foo" == "foo"` vacuously. |

## Intentional (out of scope)

- `field named field` — Rails builds a bespoke `testings` table via in-test
  `create_table`. Adding it reintroduces the per-test DDL this migration removes,
  for a column-name edge case unrelated to MySQL DDL cost.
- `partial insert off with changed composite identity primary key attribute`
  (PG-only) — pre-existing connection-pool skip, unchanged from `main`.
